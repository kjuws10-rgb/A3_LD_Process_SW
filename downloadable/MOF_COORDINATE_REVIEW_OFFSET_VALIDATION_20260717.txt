MOF Process / Review 좌표 및 물리 Offset 검증 결과
작성일: 2026-07-17

1. 기존 결과의 문제

첨부 화면의 Process 2D Matrix 값은 설정값 기준으로 정상 계산되었다.
예: B1 Process G = (-99.830, 1024.804)

하지만 기존 Review 2D Matrix는 Review Camera 좌표가 아니었다.
기존 식:

ReviewCoordinate = ProcessStage - SelectedHeadDoeStage

이 식은 선택한 H2/DOE05를 다시 원점으로 둔 Scanner/DOE 상대좌표이므로, B1이 약 (-99.560, -1024.894)로 표시됐다.
따라서 Process 좌표와 Review 좌표 사이에 Camera→Scanner 물리 Offset (210, 1060)이 반영된 모습이 보이지 않았다.

2. H1 초기위치 검증

Review Camera Center = (105, 1200) mm
Review Camera → H1 Physical Offset = (10, 1000) mm

H1 Expected Initial Stage:

(105, 1200) + (10, 1000)
= (115, 2200) mm

H2 Scanner Center:

H2.X = H1.X + ScannerPitchX
     = 115 + 200
     = 315 mm

H2.Y = H1.Y + EvenScannerYOffset
     = 2200 + 60
     = 2260 mm

Review Camera → H2 Physical Offset:

(315, 2260) - (105, 1200)
= (210, 1060) mm

3. 수정된 Review 좌표식

ScannerRelativeStage = ProcessStage - SelectedScannerCenter

ReviewCameraCoordinate
  = ScannerRelativeStage
  + CameraToSelectedScannerPhysicalOffset
  + SelectedDoeStageOffset

동일 검증식:

ReviewCameraCoordinate
  = ProcessStage - ReviewCameraCenter + SelectedDoeStageOffset

H2는 Even Head이고 DOE05의 Scanner Offset은 (-0.27, -0.09) mm이다.
Even Head 부호를 Stage 축으로 환산하면 DOE05 Stage Offset은 (-0.27, +0.09) mm이다.

5. 프로그램 개선사항

- FirstScannerInitialStageX/Y 입력 추가
- ScannerOriginTolerance 입력 추가
- H1 Initial과 ReviewCenter + PhysicalOffset 자동 일치 검증
- 이전 CSV에 H1 초기위치가 없으면 Physical Offset으로 자동 생성
- Scanner 배치는 명시적인 H1 Initial Stage를 기준으로 생성
- Review Matrix를 실제 Review Camera 좌표계로 변경
- DOE 4x4 선택 Beam의 Stage 방향 Offset 반영
- Matrix Tooltip에 Scanner 상대좌표, Physical Offset 재계산 좌표 및 일치 오차 표시
- ForwardTransportSignY 음수 입력 처리 오류 수정

6. 운전 조건

H1 초기위치와 물리 Offset 기대값이 허용오차를 초과하면 실제 장비에서는 Process Plan 생성 또는 가공 시작을 차단해야 한다.
본 WPF 예제는 Summary에 원점검증 정상/불일치를 표시하여 설정 오류를 확인하도록 구성한다.
