Review 측정 / Review Offset 구조화 가이드

0. 반드시 분리해야 하는 두 종류의 Offset
- CameraToScannerPhysicalOffset: Review Camera 광학 중심과 Scanner 가공 중심 사이의 고정 설비 Geometry
- DynamicReviewCorrection: 가공 후 측정된 오차를 다음 Process Plan에 반영하는 동적 보정값
- 물리 Offset은 Calibration ID와 장비 버전으로 관리하고, 동적 보정은 Lot/Panel/Head/Batch 이력으로 관리한다.
- 좌표식은 ScannerRelative = (ProcessStage - ReviewCenter) - CameraToScannerPhysicalOffset이다.
- 위 결과는 ProcessStage - ScannerCenter와 반드시 같아야 한다.

1. 현재 프로젝트에서 미개발인 부분
- Vision PC 또는 Review Camera에서 실제 측정 좌표를 받는 I/F
- APC 파일을 감시/다운로드/파싱하는 구성
- Fine/Rough/Simple 보정 모드별 운영 UI
- 보정값 승인, 누적, 초기화, 이력 관리 UI
- 생산용 보정 테이블 파일 저장 정책

2. 이번 base 코드에서 먼저 만든 구조
- ST_REVIEW_MEASUREMENT_POINT: 한 측정점의 Design/Process/Review/Measured/Error 좌표 보관
- ST_REVIEW_MEASUREMENT_BATCH: 한 기판 또는 한 공정에서 나온 측정점 묶음
- ST_REVIEW_OFFSET_POLICY: Gain, Max Offset, 최소 샘플 수, 누적 여부, 다음 Plan 적용 여부
- ST_REVIEW_HEAD_OFFSET: Head별 계산 Offset
- ST_REVIEW_CORRECTION_RESULT: 실제 적용 가능한지와 적용할 Offset 목록

3. 공정 적용 방식
- CStationProcess.Start()에서 가공 task 완료 후 REVIEW_MEASURE 단계 진입
- 현재는 CreateSimulationReviewBatch()가 샘플 측정값을 생성
- 실제 Vision I/F가 붙으면 이 위치에서 통신 응답 또는 파일을 ST_REVIEW_MEASUREMENT_BATCH로 변환
- REVIEW_OFFSET 단계에서 CReviewCorrectionManager.Calculate() 호출
- 적용 가능한 경우 ApplyToProcessPlan()이 Hxx_OFFSET_X/Y를 갱신
- 갱신된 Plan으로 ST_PROCESS_MODEL을 다시 만든다

4. 요구사항 대응 방향
- 0선 방어: REVIEW_MEASUREMENT_MODE=ZeroLineDefense
- Fine 보정: REVIEW_MEASUREMENT_MODE=FineCorrection, 다점 측정 기반 선형 보정 확장
- Rough 보정: REVIEW_MEASUREMENT_MODE=RoughCorrection, Cell/Head 1 Point 측정 후 일괄 Offset
- Simple 보정: REVIEW_MEASUREMENT_MODE=SimpleCorrection, 작업자 Key-in 값을 ST_REVIEW_HEAD_OFFSET으로 변환
- APC 보정: REVIEW_MEASUREMENT_MODE=ApcCorrection, APC 파일 row를 ST_REVIEW_MEASUREMENT_POINT로 파싱

6. 핵심 파라미터
- REVIEW_MEASUREMENT_USE: Review 측정 단계 사용 여부
- REVIEW_MEASUREMENT_MODE: ZeroLineDefense/Fine/Rough/Simple/APC/Remote
- REVIEW_SAMPLE_PER_HEAD: Head별 simulation 측정점 수
- REVIEW_BEAM_NO: 분기빔 측정 기준 번호, 기본 11
- REVIEW_OFFSET_GAIN_X/Y: 보정 반영 비율
- REVIEW_OFFSET_MAX_X/Y: 1회 최대 보정량
- REVIEW_OFFSET_ACCUMULATE: 기존 Hxx_OFFSET과 누적 여부
- REVIEW_OFFSET_APPLY_TO_NEXT_PLAN: 다음 Plan 반영 여부
