5. 비기능 요구사항 (Non-Functional Requirements)
- 정밀도: Align 및 가공 위치 정밀도는 설비 사양서 (Mech Spec) 를 준수해야 함.
- 안정성: 24 시간 연속 운전 시 메모리 누수 없이 안정적이어야 하며, 비정상 종료 시 복구 로직이 포함되어야 함.
- 사용자 편의성: 복잡한 보정 로직 (Fine/Rough/Simple) 은 운영자가 직관적으로 이해할 수 있는 UI와 가이드 (Flow Chart 기반) 를 제공해야 함.
- 데이터 무결성: Recipe 변경 이력, 가공 로그, 검사 데이터는 손실 없이 저장 및 관리되어야 함.
6. 납품물 및 검수 기준
1. 소스 코드 및 실행 파일: 전체 모듈 포함
2. 납품물: 설계 문서, 사용자 매뉴얼, 통신 규격서격 (ICD), 테스트 보고서
3. 검수 시나리오:
- 자동 가공 사이클 정상 수행 확인
- 각 보정 모드 (Fine, Rough, Simple) 에 따른 정밀도 향상 검증
- 인터락 발생 시 안전 정지 및 복구 테스트
- Recipe 변경 시 실제 가공 파라미터 적용 확인
- TEG 및 MOF 가공 기능 정상 동작 확인
- 상호 협의한 체크 시트를 기반으로 기능 검증

===== 가공좌표, 리뷰좌표기준점 =====
===== Default Offset =====
각 스캐너들 간에 가공된 간격에 대해 X, Y 방향에 대해 모두 등간격을 이루도록 스캐너별 Offset을 만드는 작업
Default Offset이 셋팅이 되고나면 모든 스캐너들은 이제 동작할 때 8개의 스캐너가 하나의 스캐너처럼 동작해야하기 때문에
X Y방향에 대해 Shift 조건이 발생하면 8개 모두 동일하게 Shift 동작하도록 되어야함
