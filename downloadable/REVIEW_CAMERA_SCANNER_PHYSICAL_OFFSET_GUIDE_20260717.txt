Review Camera ↔ Scanner Physical Offset 적용 가이드
작성일: 2026-07-17

1. 왜 필요한가

Review Camera가 기판의 AK 및 가공점을 보는 위치와 Scanner가 레이저를 조사하는 위치는 물리적으로 떨어져 있다.
따라서 Review Camera에서 계산한 좌표를 그대로 Scanner 명령으로 사용할 수 없으며, 두 장치 중심 사이의 고정된 거리와 방향을 반드시 적용해야 한다.

장비 배치는 기판의 정물류 진행 기준으로 Review Camera가 먼저 있고 Scanner가 뒤쪽에 있다.
기판은 Scanner 뒤쪽까지 전진한 뒤 이동 방향을 반전하며, 역물류 방향으로 돌아오면서 MOF 가공한다.
가공이 끝나면 다음에 만나는 Review Camera에서 결과를 측정하고 Home으로 복귀한다.

2. 부호 정의

CameraToScannerPhysicalOffset_i = ScannerCenter_i - ReviewCameraCenter
ScannerCenter_i = ReviewCameraCenter + CameraToScannerPhysicalOffset_i

Offset의 +X/+Y 방향은 Stage Global 좌표축의 +X/+Y 방향과 동일하게 정의한다.
실제 장비의 Stage 이동 방향과 좌표 증가 방향이 반대라면 axisSignX/axisSignY를 Calibration 데이터에 별도로 기록해야 한다.

3. Multi Scanner의 Head별 물리 Offset

H1을 기준 Calibration Head로 사용한다.

OffsetX_i = ReviewToFirstScannerOffsetX + (HeadIndex - 1) * ScannerPitchX
OffsetY_i = ReviewToFirstScannerOffsetY + EvenOffset

홀수 Head의 EvenOffset = 0
짝수 Head의 EvenOffset = EvenScannerYOffset

4. 가공좌표 변환

CameraRelativeX = ProcessStageX - ReviewCenterX
CameraRelativeY = ProcessStageY - ReviewCenterY

ScannerRelativeX = CameraRelativeX - CameraToScannerPhysicalOffsetX_i
ScannerRelativeY = CameraRelativeY - CameraToScannerPhysicalOffsetY_i

동일한 검증식:

ScannerRelativeX = ProcessStageX - ScannerCenterX_i
ScannerRelativeY = ProcessStageY - ScannerCenterY_i

위 두 계산 결과가 같지 않으면 Review Center, Scanner Center, Physical Offset 또는 축 부호 설정이 서로 일치하지 않는 것이다.

5. MOF Gx/Gy 생성

Odd Head:
Gx = -ScannerRelativeX
Gy = +ScannerRelativeY

Even Head:
Gx = +ScannerRelativeX
Gy = -ScannerRelativeY

6. 기본 예제

Review Camera Center = (105.0, 1200.0) mm
Review Camera → H1 Physical Offset = (374.7, 440.1) mm

H1 Scanner Center:
(105.0, 1200.0) + (374.7, 440.1)
= (479.7, 1640.1) mm

Scanner Pitch X가 100 mm이면 H5의 X Offset은 다음과 같다.

H5 Offset X = 374.7 + (5 - 1) * 100
            = 774.7 mm

H5 Scanner Center X = 105.0 + 774.7
                    = 879.7 mm

7. 고정 물리 Offset과 동적 Review Correction의 차이

CameraToScannerPhysicalOffset:
- 장비 조립치수 및 Calibration 결과
- Review Camera 중심과 Scanner 중심의 고정 관계
- 장비 Geometry 버전으로 관리
- Recipe, Lot, Panel이 바뀌어도 임의 변경 금지

ProcessOffsetGlobal / Dynamic Review Correction:
- 실제 가공 후 Review 측정에서 확인된 오차 보정값
- 다음 가공좌표의 정확도를 높이기 위한 Feedback 값
- Head, Lot, Panel, Batch별 이력 관리 가능
- Physical Offset과 별도 저장 및 적용

8. 운전 전 필수 검증

- Process와 Review의 calibrationId가 같은지 확인
- ReviewCenter + PhysicalOffset = ScannerCenter 검증
- Head별 Pitch 및 짝수 Head Y Offset 검증
- Stage 축 방향과 axisSignX/axisSignY 검증
- Calibration 변경 시 기존 좌표 Plan 재생성
- 허용오차 초과 시 가공 시작 금지

9. 실제 Stage 왕복 시퀀스

1) Home Stage Y에서 시작
2) 정물류 방향으로 Review Camera 위치를 먼저 통과
3) Review Camera 뒤쪽의 Scanner 가공 시작측/Turnaround 위치까지 이동
4) Stage 이동 방향 반전
5) 역물류 방향으로 이동하면서 MOF 가공
6) MOF 완료 후 Review Camera 위치에서 가공 결과 측정
7) 측정 완료 후 Home Stage Y로 복귀

기본값에서 +Y가 정물류 방향이면 다음 조건을 만족해야 한다.

ReviewCenterY < ScannerCenterY
(ScannerCenterY - ReviewCenterY) * ForwardTransportSignY > 0

MOF 실행 순서:

ForwardTransportSignY = +1이면 ProcessStageY 큰 좌표부터 작은 좌표 순서
ForwardTransportSignY = -1이면 ProcessStageY 작은 좌표부터 큰 좌표 순서

Recipe의 Row/Column 생성 순서와 실제 MOF 실행 순서는 다를 수 있으므로, ST_PROCESS_PLAN에는 MofSequence를 별도로 저장한다.
